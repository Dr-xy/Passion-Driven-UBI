提案書：情熱駆動型ベーシックインカム（PD-UBI）と個人データ主権について

1. 要旨 (Executive Summary)
現在の世界的な通貨システムは「負債」と「中央集権的な信用」の上に成り立っており、深刻な富の二極化とデータの全体主義を招いている。本提案は、全く新しい経済パラダイム――**情熱駆動型ベーシックインカム（Passion-Driven UBI）**を提示するものである。
このシステムは**デュアルトークン・メカニズム（二重トークン制）**を採用し、通貨発行権を中央銀行から個々の個人へと分散させる。その核心的なロジックは、「集団内における個人の情熱と注目」を通貨として定量化することにある。
同時に、ハードウェアレベルでの身分証明となる**パーソナルサーバー（Personal Server）**を統合することで、分散型かつ反インフレ的で、調整可能、そしてデータ主権を保障するデジタルシステムを構築する。これは「デジタル都市国家」から「個人の主権的自由」への飛躍を実現するものである。

2. コアメカニズム：PD-UBI 経済モデル
本プロトコルは、**Token A（投票権）とToken B（流通通貨）**を分離した設計を採用する。
2.1 デュアルトークンの定義
Token A（The Ballot / 投票用紙）：
	•	属性： 取引不可、譲渡不可、定期的に発行され、定期的にリセットされる。
	•	機能： 個人の「価値定義権」および「ガバナンス投票権」を象徴する。
	•	発行の起源（Origin of Issuance）：
	◦	Token Aの生成は計算能力（ハッシュレート）に依存せず、厳格に**「人間の実存」**にアンカー（紐付け）される。
	•	初期配布：
	◦	身元確認を経てシステムに参加したあらゆる参加者は、その身元が確認された瞬間からToken Aの生成が自動的にトリガーされる。急激な供給ショックを防ぐため、新規ユーザー向けのToken Aは10週間にわたり段階的に配布される。
	◦	（注：Token Aは、人間であることに付随する「社会参加権」を表す。それは後天的に獲得する資本ではなく、存在し、情熱を表現するための生まれながらの権利である。）
Token B（The Currency / 通貨）：
	•	属性： 自由に取引可能、分割可能、保管可能。
	•	機能： 市場の流通媒体であり、商品やサービスの購入に使用される。
	•	初期配分： Token Aを消費することによってのみ生成される。
2.2 情熱駆動型の配分制度 (Passion-Driven Allocation)
「すべての『いいね』は、極小規模な通貨発行である。」
通貨はもはや中央銀行によって無から印刷されるのではなく、個人の「情熱的な行為」によって鋳造される。
	•	操作： ユーザーは、自分が認める人物や物事に「いいね（投票）」をするために、1 Token Aを消費する。
	•	変換式： 1.0 Token A -> 1.5 Token B
	•	配分比率： 1 Token Aを消費した後：
	◦	1.0 Token B は、「いいね」をした本人に還付される（価値発見に参加したことに対するUBI基礎所得として）。
	◦	0.5 Token B は、被投票者に報酬として与えられる（価値を創造したことに対する社会的承認として）。

3. 経済の循環と安定性
悪性インフレを防ぎ、システムの新陳代謝を維持するため、閉ループ型の還流メカニズムを導入する。
3.1 取引税の還流 (Tax Recirculation)
	•	ルール： システム内でのToken Bの送金や取引ごとに、自動的に1%のプロトコル手数料が徴収される。
	•	行方： この税収は直接**焼却（Burn）され、同額が「公共準備プール」**に記録される。
	•	再生： 公共準備プールに蓄積された資金をアンカーとして、システムは「1.5 Token B 焼却 = 1.0 Token A 再生」というマクロ均衡式に基づき、次サイクルのToken Aを計算・鋳造し、全参加者に再配布する。
	•	意義： 通貨の流通が速ければ速いほど、UBIの発行量は増加する。通貨は単なる富の倉庫ではなく、流動するエネルギーとなる。
3.2 緩衝プールと発行平滑化メカニズム
	•	設定： システム創世記（Genesis）において、最大**30%**の追加Token Bを公共準備プールに注入・留保する。
	•	機能： 「アルゴリズム中央銀行」の準備金として機能し、**「発行平滑化アルゴリズム」**の実行に使用される。
	•	平滑化ルール： 参加者の基本所得（UBI）への期待値を安定させ、劇的な変動を避けるために：
	◦	移動平均アンカー： 週間Token A発行総量は、前週の即時税収のみに依存せず、過去10週間の税収還流の移動平均に基づくパーセンテージに従わなければならない。
	◦	変動サーキットブレーカー： システムは週間発行量の最大変動幅を、過去10週平均の ±15% に制限する。
	▪	ピークカット（削峰）： 今週の税収が急増し、算出された発行量が移動平均の15%を超えた場合、超過分は自動的に緩衝プールに預け入れられる。
	▪	ボトムアップ（填谷）： 今週の税収が暴落し、算出された発行量が移動平均の15%を下回った場合、緩衝プールから自動的に不足分が補填される。
	◦	（注：緩衝プールの蓄積が基準値の±30%を超えた場合、プロトコル手数料率を±0.25%自動調整し、市場の過熱を抑制または促進する。）

4. インフラストラクチャとアイデンティティ
分散型システムにおける「シビル攻撃（Sybil Attack）」問題を解決するためには、オープンなインターネット上で「あなたがあなたであること」を確認する必要がある。これは一つの挑戦であり、ここで「パーソナルサーバー」という概念を提案する。
4.1 パーソナルサーバー (The Personal Server)
	•	定義： 各参加者は、物理的に独立した**自己主権型サーバー（Personal Sovereign Server）を所有しなければならない。これは単なる計算機器ではなく、デジタル世界における参加者の「物理的アンカー」であり、「唯一のデジタルな肉体」**である。
	•	アイデンティティのアンカー： あなたのデジタルIDは、このサーバーのハードウェア指紋と暗号学的に紐付けられる。このサーバーは、インターネット上に存在する唯一の証明書であり、アイデンティティが虚空から生成されたり、大量複製されたりしないことを保障する。
コア機能：
	•	ハードウェア級DID： 内蔵されたセキュアエレメント（Secure Element）を利用し、偽造不可能な暗号署名を提供することで、「私は唯一の実在する人間である」ことを証明する。
	•	人間性の証明 (Proof of Humanity)： 「あなたが（AIスクリプトではなく）あなたである」ことを証明したノードのみが、Token Aの配布に参加する資格を持つ。これは、生身の人間社会とAI自動化プログラムを隔てるファイアウォールである。
	•	分散ノードマイニング： パーソナルサーバーはネットワークの分散ノードとして機能し、PD-UBIの「いいね」署名、取引検証、台帳維持を自動的に処理する。
	•	主権データ保管庫： このサーバーは個人の医療、社交、金融データをユーザーの自宅に物理的に保存する。ユーザーによって許可された外部アプリケーションにのみ開放され（プライバシー・ファースト）、権限は完全にカスタマイズ可能である。
現在の課題と移行：
	•	ハードウェアの空白： 現在、このような「個人の主権」基準を完全に満たす消費者向けサーバーやOSは存在しない。これには、個人の真正性のために設計された全く新しいハードウェアとオペレーティングシステムが必要である。
	•	コールドスタート問題： PD-UBIシステムが実行可能であると証明されるまで、過渡的な検証スキームを模索する必要がある。大量のユーザーが流入した際、専用ハードウェアなしでいかにアイデンティティの真正性を担保するかは、巨大なエンジニアリング上の課題である。
4.2 アイデンティティの危機と再構築
長い間、私たちは「身元確認」の権限を第三者機関（政府、銀行、巨大テック企業）に譲渡してきた。Web 2.0時代には、我々はビッグデータの農奴となり、プライバシーだけでなく、自己データへのコントロールさえも失った。 来るべきAI時代において、我々はさらに深刻な課題に直面している。インターネットはボットによって製造された世論で溢れかえっている。我々は大企業の手からデータ主権を取り戻す必要がある。さらに切実なのは、アルゴリズムが生み出すノイズの中から、生身の人間の発言権を取り戻すことである。
直面している問題はこうだ： 安全を保障するためにアイデンティティは完全に**自己管理（Self-Custody）されなければならず、同時に社会にアクセスするために検証可能性（Public Verifiability）**を備えていなければならない。「独立したインターネット実体」として存在しながら、世界に向かって「私は私である」と証明できるシステムを構築することは、我々が真剣に思考し解決すべき命題である。

5. ガバナンス
5.1 静的な条文ではなく、動的なコード
	•	DAOガバナンス： Token Aを獲得できる参加者はすべて投票権を持つ。
調整可能なパラメータ： システム内のすべてのパラメータは、全参加者の投票によって変更設定が可能である。
	•	主要な制御レバー：プロトコル手数料率 (The Fee Rate) プロトコル手数料は単なる税金ではなく、本システムの「マクロ調整ツール」である（伝統的金融における金利に類似する）：
	◦	インフレ対抗（冷却）： 市場が長期にわたり過熱し、物価が高騰した場合、参加者は投票により手数料率を引き上げる（例：1% -> 1.5%）ことができる。これによりToken Bの焼却を加速させ、市場の通貨供給を減らしつつ、UBI発行量を増やして生活を安定させる。
	◦	不況対抗（刺激）： 経済が長期停滞し、取引が萎縮した場合、参加者は投票により手数料率を引き下げる（例：1% -> 0.5%）ことができる。これにより取引摩擦を減らし、投資と消費を奨励し、経済回復を刺激する。
5.2 直接投票メカニズム
	•	代理人や委任メカニズムは設けず、参加者は自ら投票に参加しなければならない。
	•	暗黙の承認： 低い投票率は、現在のシステム状態に対する「暗黙の承認（Tacit Approval）」と見なされる。これは現在のパラメータが良好に機能していることを意味し、広範な問題が発生した場合にのみ、参加者は投票を通じて変革を能動的にトリガーする。

6. ビジョン (Vision)
「デジタル都市国家」から「個人の主権」へ
インターネットは大航海時代の蛮行（Web 1.0）とプラットフォーム帝国の独占（Web 2.0）を経て、今まさにインターネットの「民主時代」へと歩みを進める時が来た。PD-UBIとパーソナルサーバーを通じて、我々は誰もが自分自身をコントロールし、自分の愛するものを推進できる世界を構築したいと願っている。
「プログラマブルな社会制度」の可能性を開く
実際、もし我々が真にパーソナルサーバーによる主権を実現できたなら、我々は「改ざん不可能な社会的身分」を「インターネットの無限の利便性」の上に真に重ね合わせることができるだろう。おそらくそれは、**「プログラマブルな社会制度（Programmable Social Systems）」**への扉を開くことになるかもしれない。
最も重要な前提は、そのような個人の主権能力は、他の組織や機関によって定義されるものではなく、必ず各個人自身によってコントロールされるものでなければならないということだ。
その基礎の上で、あなたは好きな社会制度を選ぶことができる。
	•	例えば、別の経済システムを共存させることも可能だ。Token AがToken Bに変換される際、自分への還付分を取り除けば、**「UBI無しの情熱駆動型競争経済システム」**が実現する。
	•	あるいは、資源によって駆動される社会制度を試してみるのもいいだろう。
	•	もしくは、**「可愛さ」**によって駆動される社会制度などはどうだろうか？
すべては可能性の中にある。
そして、あなたは依然として荒野のインターネットの中で生きることもできる。パーソナルサーバーは強制ではない。 あなたが真実の価値を提供できる限り、たとえ物々交換の生活であっても、あなたはこれらすべての制度から離れ、何の認証もないインターネットの中で純粋に生きることができると、私は信じている。
